Revenir dans le passé avec Git

Il y a plusieurs façons de revenir dans un état antérieur avec git. Cependant, certaines sont plus dangereuses que d'autres, ou plus permanentes. En effet, certaines méthodes ne font que désengager un commit, tandis que d'autres l'effacent complétement de tout historique. On perd donc la trace de son existence.

Git log --oneline: Permet de consulter l'historique des commit et les codes.

Git checkout

Git checkout �ID�: Spectateur pour voir comment le commit était dans le temps, impossible de modifier, git checkout master pour revenir dans le présent
Git checkout �ID� nom du fichier: Modifie le fichier dans l'état qu'il était au moment du commit associé au ID, git checkout master pour revenir avant la modification du fichier. Ces commandes sont plus utiles pour la modification d'un seul fichier.

Git revert

Git revert �ID�: Défait un commit spécifique qui a été fait, peut "revert" un commit revert, l'avantage est qu'on ne perd rien dans l'historique, on peut toujour "revert" le revert.

Git reset

Git reset HEAD nom du fichier: Vide le staging
Git reset --hard: Revient au dernier commit, detruit toutes les modifications entre.
Git reset �ID�: Revient au commit sélectionner, mais laisse toutes les modifications faites depuis en staging.
Git reset HEAD^ --soft: Chaque chapeau est pour revenir en arrière de commit de un cran, le soft met toutes les modifications depuis en staging,--mixed ne met pas les modifications en staging, mais ne les détruit pas.

-git commit --amend: Rajoute des hotfixes(améliorations) sur quelque chose qu'on vient juste de commit, doit stager les nouvelles modifications et après, écrire la commande.

L'interface

Panneau de gauche

Local: Références à des branches locales permettant de séparer le travail
Remote: Références à des branches du projet hébergé sur GitHub (ou autre)
Pull requests: Montre les demandes actives pour fusionner des branches
Tags: Pointeurs vers des commits
Stash: Permet de voir les fichiers modifiés mis de côté

Menu supérieur

Undo & Redo: Permet d'annuler ou de refaire une action dans l'interface. Attention, ce n'est pas un revert commit!
Pull: Récupère et fusionne automatiquement une branche distante dans votre branche locale
Push: Envoie vos modifications (commits) sur la branche distante
Branch: Crée une copie d'une branche et permet de travailler à côté de celle-ci
Stash: Permet de mettre de côté des changements (//WIP) en créant des copies des fichiers sur la machine locale
Pop stash: Permet de rouvrir des modifications (//WIP) mises de côté

Panneau commits

Situé à droite de la fenêtre, il contient les fichiers et les changements apportés à une branche.

Open

Permet d'ouvrir un projet déjà sur sa machine
File > Open > Choisir le dépôt à ouvrir

Clone

Permet de créer une copie d'un projet local ou d'un projet hébergé sur un site comme GitHub
File > Clone > Entrer l’URL du projet ou le chemin
Nécessite l’accès au dépôt si le projet est privé

Init

Permet de créer un nouveau projet
File > Init
En local > Choisir le dossier qui contiendra le projet
Sur GitHub > Choisir son compte GitHub, donner un nom > Créer et cloner
